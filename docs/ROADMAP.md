## 贡献指南

1. 每个功能开发前，先创建issue描述需求
2. 从 `feature/cogalpha-mvp` 分支创建子分支
3. 编写对应测试后再实现功能
4. 确保所有测试通过后再提交PR
5. 更新ARCHITECTURE.md和ROADMAP.md

## 已知限制

- MVP仅使用合成数据，结果不代表真实市场表现
- 突变和交叉操作为stub，尚未实现完整LLM驱动的进化
- 不支持TA-Lib技术指标（使用pandas/numpy fallback）
- 不支持多因子非线性融合因子的实际生成（仅模板就绪）
