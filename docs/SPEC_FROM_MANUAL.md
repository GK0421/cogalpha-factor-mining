# CogAlpha MVP 需求规格书（从操作手册提取）

> 来源：《CogAlpha全自动因子挖掘框架：从零部署到样本外验证的完整操作手册》
> 提取时间：2026-06-19

## 1. 项目目标

构建一个全自动化的量化因子挖掘框架，利用大语言模型（LLM）生成、演化、评估Alpha因子，最终输出经过严格样本外验证的精英因子库。

## 2. 数据要求

### 2.1 核心字段
- `open`（开盘价）
- `high`（最高价）
- `low`（最低价）
- `close`（收盘价）
- `volume`（成交量）

### 2.2 时间范围
- **全量数据**：2011-01-01 至 2025-12-31
- **训练期**：2011-01-01 至 2019-12-31（用于因子生成、质量检查、适应度评估）
- **样本外测试期**：2020-01-01 至 2025-12-31（仅用于最终验证，严禁在进化中使用）

### 2.3 数据清洗规则
- 剔除ST、*ST股票
- 剔除当日涨跌停股票
- 剔除上市未满1年新股
- 缺失值前向填充（forward fill）
- 序列开头无法填充的缺失值删除

## 3. 七级智能体架构（21个Agent）

| 层级 | 主题 | 智能体数量 | 核心金融视角 |
|------|------|-----------|-------------|
| Lv1 | 市场结构与周期 | 3 | 长期趋势、市场阶段、周期性状态转换 |
| Lv2 | 极端风险与脆弱性 | 3 | 尾部风险、崩溃前兆、系统脆弱性模式 |
| Lv3 | 量价动态 | 3 | 流动性、订单失衡、量价一致性 |
| Lv4 | 价格波动行为 | 3 | 趋势持续性、短期反转、波动聚集 |
| Lv5 | 多尺度复杂性 | 3 | 分形粗糙度、回撤恢复几何、长记忆特征 |
| Lv6 | 稳定性与机制控制 | 3 | 时间稳定性、自适应控制、信号激活机制 |
| Lv7 | 几何与融合 | 3 | K线形态识别、多因子线性/非线性融合 |

## 4. 因子生成流程

1. **首轮生成**：21个智能体 × 8个初始因子 = 168个候选因子
2. **提示词组装**：读取Jinja2模板，动态注入生成模式、无效案例摘要、精英因子思路
3. **并发调用**：使用ThreadPoolExecutor或类似机制并发调用LLM
4. **代码提取**：从LLM返回中提取 `<function N>...</function N>` 包裹的代码
5. **降级解析**：若XML标签未命中，使用正则匹配 `def` 开头的函数

## 5. 质量检查流水线

### 5.1 代码质量检测（语法与静态扫描）
- `ast.parse` 语法检查
- 变量与引用检查（未定义变量检测）
- 基础未来函数扫描：`shift(-N)`, `iloc[-1]`, `tail(1)` 等

### 5.2 代码修复（自动纠错）
- 受控沙盒执行，捕获运行时异常
- 调用LLM进行修复（仅修复局部编码错误，不改变核心逻辑）
- 验证修复后的代码通过语法检查

### 5.3 语义评估（逻辑一致性）
- 文档字符串与实际代码的逻辑一致性对比
- 数学运算技术正确性判断
- 经济可解释性审查
- **深度未来信息泄漏审查**：时序截断单元检测

### 5.4 逻辑提升（优化与简化）
- 复杂度简化
- 逻辑澄清（添加注释）
- 冗余消除
- 保持原始经济含义不变

## 6. 未来信息泄漏检测（时序截断单元）

### 核心算法
1. 随机抽取多个测试日期 T
2. 准备全量数据 D_full 和截断至日期 T 的数据 D_truncated
3. 分别运行因子代码，得到两列因子值
4. 比较日期 T 的数值
5. 若差异 < 1e-10 且相关系数接近1，判定无泄漏；否则标记 suspected_leakage

### 常见陷阱
- 误用 `shift(-1)` 获取下一期数据
- 滚动计算未执行 `shift(1)` 对齐
- 财务数据未按公告日期对齐

## 7. 适应度评估与精英筛选

### 7.1 核心指标
- **IC**：因子值与下一期收益率的线性相关性
- **ICIR**：IC均值/IC标准差，衡量稳定性
- **RankIC**：斯皮尔曼秩相关系数，对异常值稳健
- **RankICIR**：RankIC的稳定性指标

### 7.2 分级标准
- **合格因子**：综合得分 ≥ 65%分位，IC≥0.005, ICIR≥0.05, RankIC≥0.005, RankICIR≥0.05
- **精英因子**：综合得分 ≥ 80%分位，IC≥0.01, ICIR≥0.1, RankIC≥0.01, RankICIR≥0.1

### 7.3 精英保留机制
- 每代保留Top 2精英因子直接复制到下一代父代池

## 8. 思维进化循环

### 核心操作
- **突变**（mutation_rate=0.4）：对单个因子微调改进
- **交叉**（crossover_rate=0.3）：融合两个父代因子逻辑
- **全新因子注入**（injection_interval=2）：每2代注入全新因子，防止局部最优

### 子代处理
所有子代必须经过质量检查流水线 → 适应度评估 → 分类（无效/合格/精英）

### 种群更新
- 新父代池 = 本代合格子代 + 上代Top 2精英

## 9. 样本外验证

### 核心原则
- **数据隔离**：样本外数据严禁在因子生成和进化中使用
- 在完全隔离的测试期（2020-2025）上重新计算因子值和绩效指标
- 对比训练期与样本外绩效，观察是否存在显著衰减

### 稳健性检验
1. 分年度/分市场状态测试
2. 参数敏感性测试（窗口参数微扰动）
3. 因子相关性分析与聚类
4. 未来信息泄漏复查

## 10. Git/安全约束

- `config.yaml` 和 `.env` 必须加入 `.gitignore`
- 通过 `config.example.yaml` 提供模板
- 严禁提交真实API Key、真实行情数据、大型缓存、中间运行产物
- 使用 `config.example.yaml` 作为无密钥的公共模板

## 11. MVP 范围声明

本MVP实现以下核心能力：
- 配置管理（YAML + 环境变量）
- 合成数据生成与验证
- 21个智能体注册与提示词模板
- Mock LLM Provider（默认）+ OpenAI兼容Provider（可选）
- 因子解析、质量检查、泄漏检测
- 因子评估（IC、RankIC、分组收益、long-short）
- 样本外区分
- CLI与一键运行
- 完整测试覆盖

**以下能力为MVP之外的未来扩展**：
- 真实A股数据接入
- 真实LLM调用（DeepSeek/Qwen）
- 完整进化循环（突变、交叉、注入）
- 参数敏感性分析
- 因子相关性聚类
- 实盘验证
